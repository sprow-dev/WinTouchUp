# CONTRIBUTING.md
Contribution guidelines.

I hope to see some code soon! If you know anything about me, you know that I am ridiculously busy and very protective
of my time. The fact that you might take time to actually help me by writing code is very important and I can't thank you enough.

## Environment setup
Clone the repo, open WinTouchUp.slnx in the latest version of Visual Studio (2022+) with .NET Desktop Application Development (.NET 10, WinForms enabled) toggled on.

That's all!

## Code Quality Expectations
I expect code that is reasonably exempt from:
Crashes, errors, warnings, cosmetic code violations, UX-degrading slowness, and otherwise.
You must deem the code good enough that it would take at most 10 minutes for the MAIN DEVELOPER to fix the code.
That would generally be if you fixed all the listed issues before submitting.
Almost always, I will nitpick the code. However, if you just fix the listed issues, I may take more time if I have enough available to me.

I will let you know if your code does not meet standards and spend up to 10 minutes attempting to fix it. (Or more if exempt. I will tell you if this applies and if I went overtime.)
If I run out of time, it is your responsibility to finish what I started. I will let you know what you need to do to complete the CQ review.

I generally handle these (PRs are priority) during lunch break so I have very tight timing. That's my reason for the 10 minute rule.

## Respect
Please do not harass or demean anybody. Follow CODE_OF_CONDUCT.md **at all times** or you will be disciplined according to said policy.

Just because you contributed code doesn't mean that it is now your codebase. Do not treat it as such.

If you have formal complaints to make about my code quality or any work done in the codebase, please open an issue and **use the appropriate template**.
Contributing code is not the place for ranting. However, I will not control code comments unless they are direct code of conduct violations.
I do ask that you are respectful with the code comments but I will not enforce it unless otherwise said previously. I also ask that you avoid profane language when commenting your code.
If you would like to make helpful comments, feel free to make them. Just don't over-comment.

## AI
I would ask that you do not have AI write more than 50% of the code that you contribute. However, I can't enforce this (for obvious reasons)
If you already wasted resources on AI, I will not stop you from having your contribution. You can't un-spend resources, after all. I do not
care about other uses of AI, like looking up code help.
**The catch**: I will NOT condone AI code that does not follow my standards. If you are already lazy enough to use AI to write all your code for you,
being so lazy that you can't get it to at least write quality code, I will immediately close your issue with no warning.

## Contact
I have a contact form in my readme. Please view readme to access the contact form and anonymously let me know if you need anything else.
However, if you want to make an edit to this file, please just make a pull request and that can be used as a formal editing channel.

I believe that everyone contributing will make good choices. The first one being helping me out so much. Every line of code is monumental.
My guidelines are just here to filter out people who don't know what they are doing, but if you've read this far, you know what you are doing.
